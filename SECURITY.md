# Security Policy

## Reporting a Vulnerability
Please report security issues responsibly.

- **Do not** open public issues that include exploit details, secrets, tokens, or private data.
- Prefer a private channel first (maintainer contact / private disclosure).

If you cannot find a private channel, open a minimal public issue **without sensitive details** and request a private follow-up.

## Supported Versions
We generally support:
- `main` (current development line)
- Recent tagged releases (when applicable)

Security fixes may be backported when the risk is high and the fix is low-risk.

## Responsible Disclosure
We appreciate coordinated disclosure and will acknowledge valid reports when appropriate.
