# Ambi-cha-so Contract v1 (Stopless / No-Exit / Low-CPU)

## 最優先ゴール

* **ターミナル（親）を絶対に落とさない**
* **停止は終了コードではなくログとSTOPフラグで制御する**
* **重い処理は分割し、CPU100%張り付き・フリーズを避ける**

## 絶対禁止（全面禁止）

* シェル：exit / return 非0 / set -e / set -euo pipefail / trap ... EXIT 等（終了コード依存・強制終了につながるもの）
* Python：sys.exit / raise SystemExit / assert / 未捕捉例外で停止
* argparse：引数エラーによる SystemExit を禁止。使う場合は捕捉して ERROR: に変換すること
* “成功/失敗”を終了コードで表現する設計（常にログで判定）

## 必須ログ規約（嘘をつかない）

* 出力は **OK:** / **ERROR:** / **SKIP:** のみを基本とする
* **SKIP は理由を必ず1行**（未来の監査ログ）
* **ERROR は「プロセス終了」ではなく「その手順の停止」**
  * 例：ERRORが出たら STOP=1 にし、以降のステップは SKIP にする（理由1行）
* 例外詳細が必要なら、標準出力に垂れ流さず **ファイルに保存**し、標準出力は1行だけ
  * 例：ERROR: exception (see out_dir/traceback.txt)

## 設計出力の型（超精密）

* **Plan.md**：疑似コード。必ず if/else/for/continue/try/catch/ERROR/SKIP/STOP を明示して、止まらない設計にする
* **Task.md**：チェックボックスで **実行順序を固定**。重い処理は「任意/最後/別ステップ」に分離
* for は探索（候補パス/ブランチ名/ファイル存在確認）に使う
  * 見つけたら break / 見つからなければ continue

## CPU・フリーズ対策（超重要）

* “重い処理”は **1ステップ1本**に分割（例：make test、reviewpack verify-only 等）
* 可能なら **smoke（軽）** と **heavy（重）** を分離し、普段は smoke のみ
* 固まりそうなら **中断しても良い**（ただし中断理由＋直前の観測結果を1行残す）
* 長時間処理は「手動実行の最後」に追いやる（自動ループには入れない）

## 判定ルール

* スクリプトは基本 **常に 0 を返す**（停止判断はログで行う）
* 次ステップへ進むかどうかは **STOPフラグ**で決める（終了しない）
